# ProVAI UI Vision

This document is the single source of truth for the ProVAI user experience. It defines our core philosophy, our guiding principles, the user journey, and the personas we are building for.

## 1. Guiding Philosophy: The Personalized AI Tutor

ProVAI is not just a tool; it is a partner in the educational journey. Its identity is defined by three core principles that guide its relationship with its users and their data.

-   **ProVAI is a *Reinforcement*, not a Replacement:** It amplifies a teacher's effectiveness and reinforces their instruction.
-   **ProVAI is a *Practice Field*, not an Exam Hall:** It provides a private, safe space for students to learn without fear of judgment.
-   **ProVAI is an *Insight Engine*, not a Surveillance System:** It generates actionable signals to help people learn, with a strict focus on privacy.

## 2. Design Principles

Every component and interaction must adhere to these three principles:

1.  **Clarity over Clutter:** The interface must be minimalist, intuitive, and focused.
2.  **Guidance over Prescription:** The application is a gentle guide, not a rigid taskmaster.
3.  **Serenity over Noise:** The experience must be calm and free of distractions to promote deep focus.

## 3. High-Level User Flow

The user's journey is a simple, gated flow that separates unauthenticated visitors from authenticated users.

1. **Unauthenticated:** A new user arrives at the **Landing Page**. From here, they can navigate to the **Authentication Page** to either Log In or Register.
2. **Authentication:** Upon successful login, the user is redirected to their personal, role-based **Dashboard**.
3. **Authenticated:** Once logged in, the user can navigate between their **Dashboard** and the main **Chat View** using a persistent navigation bar. Logging out returns them to the Landing Page.
